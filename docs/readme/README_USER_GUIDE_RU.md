# Руководство пользователя

Этот файл для обычного игрока, которому нужно просто поставить `StateHelper` и запустить его без ручной возни с кучей папок.

## Быстрая установка

1. Положи [StateHelper.lua](../../StateHelper.lua) в папку `moonloader`.
2. Запусти игру.
3. Если папки `StateHelper/` рядом еще нет, скрипт сам скачает нужные файлы с GitHub.
4. После установки скрипт либо перезагрузится сам, либо понадобится обычный перезапуск игры.

## Как это работает

- игроку нужен только один `Lua`-файл
- остальная структура проекта скачивается автоматически
- при следующих запусках скрипт сам проверяет обновления
- актуальная версия проекта сейчас идет по ветке `v5.0.0-alpha`

## Что обычно скачивается

- bootstrap-модули
- core-модули
- пользовательские фичи
- механики
- пакеты фракций
- служебные `json`
- шрифты и картинки из `StateHelper/Arizona_data/remote`

## Как работает обновление

- при запуске скрипт сверяет локальную версию с `update_info.json`
- если доступна новая версия, он может поставить ее автоматически
- если автообновление отключено, скрипт просто покажет, что есть новая версия

## Если что-то пошло не так

- проверь, что `MoonLoader` установлен нормально
- проверь, что игре ничего не мешает выйти в сеть
- не удаляй вручную `StateHelper/Arizona_data/installed/managed_files.lst`
- если первая установка или обновление сорвались, просто перезапусти игру и дай скрипту повторить загрузку
- если нужно понять, как разложена текущая установка, можно использовать `tools/repair-live-install.ps1`

## Что не нужно делать руками

- не нужно самому раскладывать модули по подпапкам
- не нужно править `manifest.lua` или `update_info.json`
- не нужно скачивать проект по отдельным `raw`-ссылкам

## Что полезно помнить

- основной вход для игрока - [StateHelper.lua](../../StateHelper.lua)
- папка `StateHelper/` создается автоматически при первой установке
- папка `State Helper/` может существовать рядом как legacy-хранилище данных
- дальше проект уже работает из своей локальной структуры
